Experimental AI Journal - Reporting Guideline

The Experimental AI Journal is commited to ensure the transparency and reproducibility of submitted AI research. Therefore, any contributions to the Journal need to ensure that all datasets, configurations, and results are adequately documented. Researchers aiming to publish in the Experimental AI Journal are asked to document their AI workflows as follows.

Specify the following information:

  • Dataset including its source, size and features
  • All data preparation steps performed
  • Data partitioning method and size of test and train sets
  • Applied ML algorithms including their settings and hyperparameters
  • Performance comparison of created ML models or baseline models

Reproducibility Report for the AIce Cream Project

This report contains the implementation details for the AutoML models created as part of the AIce Cream research project. It is structured to fulfill the reporting requriments outlined by the guidelines of the Experimental AI Journal.

Dataset

Data file Ice_Cream_Sales.csv
Source Collected as part of AIce Cream project
Size 3698 rows
Features Temp_C, Is_Weekend, Is_Holiday, Rainfall_mm, Flavour, Sales_EUR

Data Preparation

Missing Value Handling Replaced missing numerical values with mean and missing categorical values with most frequent value.
Outlier Handling Detected outliers using Z-scores and dropped rows containg any outliers.
Feature Engineering Dropped zero-variance features and features with correlation coefficient higher than 0.95

Model Training

Training Objective The objective was to predict the features "Sales_EUR" using the optimization metric "Adjusted R^2"
Trained Models Linear Regression, Random Forest, Gradient Boosted Trees, XGBoost Tree Ensemble
Data Partitioning Data was sampled randomly with a 30% test set ratio and using the random seed "481357"

Model Performance

Model Adjusted R^2
Linear Regression 0.886
Random Forest 0.737
Gradient Boosted Trees 0.873
XGBoost Tree Ensemble 0.945
